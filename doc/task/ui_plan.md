# แผนการสร้าง UI ทดสอบ API และระบบ Monitoring

ตามที่คุณต้องการให้ "ออกแบบมาก่อน" สำหรับหน้าเว็บทั้ง 2 ส่วน (หน้าทดสอบแปลภาษา และหน้า Monitor แบบ Real-time) ผมได้นำหลัก **Ponytail Principle** มาวิเคราะห์เพื่อหาทางออกที่เรียบง่ายและคุ้มค่าที่สุดใน Phase ปัจจุบันครับ

## 📝 1. หน้าต่างทดสอบแปลภาษา (Translation Tester UI)

**ปัญหา:** ต้องการหน้าเว็บสำหรับกรอกข้อความภาษาไทย แล้วยิง API เพื่อขอแปลเป็นภาษาอังกฤษ
**แนวทาง (Option B - Minimal that works):** 
สร้างไฟล์ `tester.html` แฟ้มเดียวที่รวม HTML, CSS (ตกแต่งให้ดูทันสมัย) และ Vanilla JavaScript 
- ผู้ใช้กรอกข้อความภาษาไทย -> กดปุ่มแปล -> โค้ด JS จะใช้คำสั่ง `fetch` ยิงไปยัง `POST /translate/` ของ FastAPI
- ระบบจะแสดง `Task ID` ที่ได้รับกลับมาบนหน้าจอ

**งานฝั่ง Backend ที่ต้องเพิ่ม:** 
เราจำเป็นต้องแก้ไขโค้ด `app/main.py` เล็กน้อยเพื่อเพิ่ม **CORS Middleware** (Cross-Origin Resource Sharing) ไม่อย่างนั้นเบราว์เซอร์จะบล็อกไม่ให้หน้าเว็บ HTML ยิง API ไปยัง FastAPI ได้

## 📊 2. หน้าเว็บ Monitor แบบ Real-time

**ปัญหา:** ต้องการหน้าเว็บเพื่อดูสถานะของระบบหรือคิวงานแบบ Real-time แต่ยังไม่รู้จะใช้อะไรทำดี

เพื่อให้สอดคล้องกับ **Ponytail Principle** ผมขอเสนอ 3 ทางเลือกสำหรับการทำ Monitoring ดังนี้ครับ:

### Option A: Zero-code / Native (แนะนำสำหรับ Phase 1 ตอนนี้)
**ใช้ RabbitMQ Management UI ที่มีอยู่แล้ว**
- **วิธีทำ:** เข้าไปที่ `http://localhost:15672` (ใช้ user/pass: guest)
- **ทำไมถึงดี:** คุณสามารถดูปริมาณข้อความที่รออยู่ในคิว (Queue), ดูอัตราการรับ-ส่งข้อมูล (Message rates), และกราฟแบบ Real-time ได้ทันทีโดยไม่ต้องเขียนโค้ดเพิ่มแม้แต่บรรทัดเดียว เหมาะมากสำหรับดูว่า API โยนงานเข้าคิวสำเร็จไหม

### Option B: Minimal HTML Polling (ถ้าอยากได้หน้าเว็บแยกจริงๆ)
**สร้าง `monitor.html` พร้อม JS Polling**
- **วิธีทำ:** สร้างหน้าเว็บง่ายๆ และเขียน JavaScript ให้คอยยิง API ไปถามเซิร์ฟเวอร์ทุกๆ 2 วินาที (Polling) ว่า "ตอนนี้คิวงานเป็นยังไงบ้าง?"
- **ข้อจำกัด:** ต้องไปเขียน API Endpoint เพิ่มใน FastAPI เพื่อดึงข้อมูลคิวส่งกลับมาให้หน้าเว็บ

### Option C: Proactive Idea (สำหรับระบบเต็มรูปแบบในอนาคต)
**ใช้ WebSockets (FastAPI + React/Vue)**
- **วิธีทำ:** เปิดช่องทางเชื่อมต่อ WebSockets ให้ Backend ดันข้อมูล (Push) มาบอก Frontend ทันทีที่มีการเปลี่ยนแปลง
- **ข้อจำกัด:** เขียนโค้ดซับซ้อนเกินไปสำหรับตอนนี้ และต้องรอนำระบบ AI หรือ Agent เข้ามาต่อท้ายก่อนถึงจะเห็นประโยชน์ชัดเจน
