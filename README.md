# 🚀 ยินดีต้อนรับสู่โปรเจกต์ HermesTranslate (คู่มือสำหรับนักพัฒนา)

สวัสดีนักพัฒนา (Developer)! 👋 ยินดีต้อนรับสู่โปรเจกต์ **HermesTranslate** ระบบแปลภาษาและจัดการเนื้อหาอัตโนมัติด้วยพลังของ AI ที่ออกแบบมาเพื่อรองรับงานจำนวนมากพร้อมๆ กัน 

ระบบนี้ไม่ได้ทำงานแบบรอรับผลลัพธ์หน้าจอตรงๆ แต่เราใช้สถาปัตยกรรมแบบ **Asynchronous + Message Queue** ควบคู่ไปกับการใช้ระบบ Multi-Agent (AI 3 ตัวทำงานร่วมกัน) เพื่อความเสถียรและแม่นยำสูงสุด

---

## 🗺️ แผนที่การเดินทาง (How to get started)

เพื่อไม่ให้คุณหลงทางในโค้ดจำนวนมาก ขอให้คุณเริ่มต้นอ่านเอกสารและทำงานตามลำดับต่อไปนี้อย่างเคร่งครัด:

### Step 1: เข้าใจภาพรวมของระบบ (The Big Picture)
ก่อนเขียนโค้ดบรรทัดแรก ให้คุณเข้าไปอ่านไฟล์ **[srs.md](srs.md)** เพื่อทำความเข้าใจว่าลูกค้าต้องการอะไร สถาปัตยกรรมระบบหน้าตาเป็นอย่างไร และระบบทำงานเชื่อมโยงกันอย่างไร

### Step 2: เริ่มต้นลงมือทำตาม Task List (Phase 1-4)
งานของคุณถูกแบ่งออกเป็น 4 เฟสย่อย (Phase) เพื่อให้คุณสามารถค่อยๆ สร้างระบบขึ้นมาทีละชิ้น ขอให้ทำตามลำดับต่อไปนี้:

1. 🟢 **Phase 1: รับงานและจัดคิว** -> เปิดไฟล์ **[hermes_task_list.md](hermes_task_list.md)** (ตั้งค่า FastAPI สร้าง Endpoint และโยนงานลง RabbitMQ)
2. 🔵 **Phase 2: ฐานข้อมูลและสมอง** -> เปิดไฟล์ **[knowledge_base_task.md](knowledge_base_task.md)** (ทำ PostgreSQL, API แอดมิน, และลอจิกคัดกรองคำด้วย Aho-Corasick)
3. 🟠 **Phase 3: ระบบ AI Agents** -> เปิดไฟล์ **[multi_agent_task.md](multi_agent_task.md)** (เขียน Worker ดึงงานจากคิว และคุม AI 3 ตัวให้ทำงานประสานกัน)
4. 🟣 **Phase 4: ประกอบร่างระบบ** -> เปิดไฟล์ **[integration_task.md](integration_task.md)** (รวบรวมทุกระบบมารันบน `docker-compose.yml` และทำ API เช็คผลลัพธ์)

> **💡 คำแนะนำ:** ภายในไฟล์ Task List แต่ละไฟล์ จะมีหัวข้อ **"ข้อควรระวังสำหรับ Junior (Gotchas!)"** ซ่อนอยู่ด้านล่างสุด ขอให้อ่านส่วนนั้นให้เข้าใจก่อนเริ่มเขียนโค้ด เพราะมันคือจุดที่คนมักจะเขียนบั๊กกันบ่อยที่สุด!
